# 12306 Ticket Skill 🚄

> 12306火车票一站式检索工具 — 直达查询 + 智能中转 + 自动解析票价

## 功能

- **一站式查询** — 直达 + 中转同命令一次性输出，自动解析票价与余票
- **智能中转** — `all`模式从直达车途经站API提取中间枢纽，只查沿线大站，精准高效
- **多线程并发** — ThreadPoolExecutor 5路并发，4-20s 完成（vs 旧版 90s+）
- **过夜换乘** — 自动识别跨日衔接（如22:30到→次日07:38发），`--max-gap 1440`启用
- **同城跨站** — 自动配对同城不同站（徐州↔徐州东、南京↔南京南），不漏方案
- **票价解析** — 从12306原始API响应中自动解析各席别票价，无需额外接口
- **三层容错** — `_get`重试 + `query_tickets`兜底 + CLI入口session刷新，防查询失败
- **紧凑输出** — 单行表格格式，一目了然

## 快速开始

```bash
# 一站式查询（直达+中转）
python3 assets/query12306.py all -d YYYY-MM-DD -f 出发站 -t 到达站

# 示例
python3 assets/query12306.py all -d 2026-07-01 -f 杭州 -t 昆明
python3 assets/query12306.py all -d 2026-07-01 -f 杭州 -t 昆明 --hubs 长沙,南昌,武汉
```

## 命令

| 模式 | 命令 | 说明 |
|------|------|------|
| 一站式 | `all -d 日期 -f 起点 -t 终点` | 直达+中转一次输出 |
| 仅直达 | `direct -d 日期 -f 起点 -t 终点` | 只看直达 |
| 仅中转 | `transfer -d 日期 -f 起点 -t 终点` | 只看中转 |
| 站点查询 | `stations -f 关键词` | 模糊搜索车站 |

## 参数

| 参数 | 说明 | 默认 |
|------|------|------|
| `-d, --date` | 日期 YYYY-MM-DD | 必填 |
| `-f, --from` | 出发站（中文/拼音） | 必填 |
| `-t, --to` | 到达站 | 必填 |
| `--hubs` | 中转候选站（逗号分隔） | 80+枢纽全量检索 |
| `--min-gap N` | 最小换乘等待（分钟） | 20 |
| `--max-gap N` | 最大换乘等待（分钟） | 360 |
| `--max-results N` | 最大方案数 | 10 |
| `--json` | JSON 输出 | 否 |

## 技术要点

- **纯Python3**，无第三方依赖，标准库即可运行
- **并发架构**：`ThreadPoolExecutor` 5路并行处理枢纽，每线程独立`Session`
- **智能枢纽提取**：`all`模式从最快直达车调用`czxx/queryByTrainNo` API获取途经站，与全国主要枢纽取交集
- **同城跨站配对**：同城枢纽（如徐州+徐州东）的l2查询覆盖所有变体，跨站组合不遗漏
- **过夜换乘**：`gap < 0`时自动`+1440`修正，配合`--max-gap 1440`支持次日早班衔接
- **过头过滤**：第1程耗时≥直达最快×0.85时判定枢纽过头，自动跳过
- **三层容错**：`_get` 3次重试 → `query_tickets` 2次兜底 → CLI入口强制刷session
- **全量收集再排序**：所有hub查完后统一按总耗时排序取top N

## 注意事项

1. 仅查询未来15天内车票
2. `all`模式通过直达车途经站智能选枢纽，通常4-20s完成；直达失败时回退16个主要枢纽
3. 查过夜换乘需加`--max-gap 1440`（24小时），默认360分钟仅覆盖同日换乘
4. 站名支持中文/拼音模糊匹配
5. 中转方案建议用`--hubs`指定关键枢纽聚焦结果
